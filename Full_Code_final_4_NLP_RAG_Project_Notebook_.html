Problem Statement

 

Business Context

 
The healthcare industry is rapidly evolving, with professionals facing increasing challenges in managing vast volumes of medical data while delivering accurate and timely diagnoses. The need for quick access to comprehensive, reliable, and up-to-date medical knowledge is critical for improving patient outcomes and ensuring informed decision-making in a fast-paced environment.

Healthcare professionals often encounter information overload, struggling to sift through extensive research and data to create accurate diagnoses and treatment plans. This challenge is amplified by the need for efficiency, particularly in emergencies, where time-sensitive decisions are vital. Furthermore, access to trusted, current medical information from renowned manuals and research papers is essential for maintaining high standards of care.

To address these challenges, healthcare centers can focus on integrating systems that streamline access to medical knowledge, provide tools to support quick decision-making, and enhance efficiency. Leveraging centralized knowledge platforms and ensuring healthcare providers have continuous access to reliable resources can significantly improve patient care and operational effectiveness.
 
Common Questions to Answer

1. Diagnostic Assistance: "What are the common symptoms and treatments for pulmonary embolism?"

2. Drug Information: "Can you provide the trade names of medications used for treating hypertension?"

3. Treatment Plans: "What are the first-line options and alternatives for managing rheumatoid arthritis?"

4. Specialty Knowledge: "What are the diagnostic steps for suspected endocrine disorders?"

5. Critical Care Protocols: "What is the protocol for managing sepsis in a critical care unit?"
 

Objective

 
As an AI specialist, your task is to develop a RAG-based AI solution using renowned medical manuals to address healthcare challenges. The objective is to understand issues like information overload, apply AI techniques to streamline decision-making, analyze its impact on diagnostics and patient outcomes, evaluate its potential to standardize care practices, and create a functional prototype demonstrating its feasibility and effectiveness.
 

Data Description

 
The Merck Manuals are medical references published by the American pharmaceutical company Merck & Co., that cover a wide range of medical topics, including disorders, tests, diagnoses, and drugs. The manuals have been published since 1899, when Merck & Co. was still a subsidiary of the German company Merck.

The manual is provided as a PDF with over 4,000 pages divided into 23 sections.

def generate_rag_response(user_input, k=5, max_tokens=400, temperature=0, top_p=0.95, top_k=50, search_type="similarity"):
    global qna_system_message, qna_user_message_template, vectorstore

    # Retrieve relevant document chunks
    local_retriever = vectorstore.as_retriever(search_type=search_type, search_kwargs={"k": k})
    relevant_document_chunks = local_retriever.invoke(user_input)
    context_list = [d.page_content for d in relevant_document_chunks]

    # Combine document chunks into a single context
    context_for_query = ". ".join(context_list)

    # Build the final prompt using the system message and user message template
    user_message = qna_user_message_template.format(context=context_for_query, question=user_input)
    prompt = f"[INST] {qna_system_message}\n\n{user_message} [/INST]"

    # Generate the response
    try:
        model_output = lcpp_llm(
            prompt=prompt,
            max_tokens=max_tokens,
            temperature=temperature,
            top_p=top_p,
            top_k=top_k,
            stop=["[INST]"],
            echo=False
        )
        response = model_output['choices'][0]['text'].strip()
    except Exception as e:
        response = f"Sorry, I encountered the following error: \n{e}"

    return response

def generate_ground_relevance_response(user_input, k=5, max_tokens=300, temperature=0, top_p=0.95, top_k=50):
    global qna_system_message, qna_user_message_template, vectorstore
    global groundedness_rater_system_message, relevance_rater_system_message, user_message_template

    # Step 1: Retrieve relevant document chunks
    local_retriever = vectorstore.as_retriever(search_type="similarity", search_kwargs={"k": k})
    relevant_document_chunks = local_retriever.invoke(user_input)
    context_for_query = ". ".join(d.page_content for d in relevant_document_chunks)

    # Step 2: Generate the RAG answer
    user_message = qna_user_message_template.format(context=context_for_query, question=user_input)
    qa_prompt = f"[INST] {qna_system_message}\n\n{user_message} [/INST]"

    qa_output = lcpp_llm(
        prompt=qa_prompt,
        max_tokens=max_tokens,
        temperature=temperature,
        top_p=top_p,
        top_k=top_k,
        stop=["[INST]"],
        echo=False
    )
    answer = qa_output["choices"][0]["text"].strip()

    # Step 3: Build the evaluation prompts using the rater system messages
    eval_message = user_message_template.format(context=context_for_query, question=user_input, answer=answer)
    groundedness_prompt = f"[INST] {groundedness_rater_system_message}\n\n{eval_message} [/INST]"
    relevance_prompt = f"[INST] {relevance_rater_system_message}\n\n{eval_message} [/INST]"

    # Step 4: Get groundedness rating
    groundedness_output = lcpp_llm(
        prompt=groundedness_prompt,
        max_tokens=150,
        temperature=0,
        top_p=top_p,
        top_k=top_k,
        stop=["[INST]"],
        echo=False
    )
    groundedness_eval = groundedness_output["choices"][0]["text"].strip()

    # Step 5: Get relevance rating

        prompt=relevance_prompt,

        top_k=top_k,
        stop=["[INST]"],
        echo=False
    )
    relevance_eval = relevance_output["choices"][0]["text"].strip()

    return answer, groundedness_eval, relevance_eval

Core Operational Insights




RAG is Critical for Clinical Precision:

Standard language models operating on pre-trained knowledge provide generic, directional answers that lack clinical utility. Grounding the pipeline via RAG introduces high-fidelity, actionable data—such as specific diagnostic signs, fluid resuscitation targets, and precise dosing guidelines—making it a mandatory baseline for responsible healthcare deployment.

Prompt Engineering Enhances Structure, Not Factuality:

While structured, role-based prompts successfully enforce safety guardrails and optimize formatting, they cannot compensate for an absent knowledge base. To achieve compliance and factual accuracy, structured prompting must be deployed alongside a robust retrieval architecture rather than as a substitute.

Retrieval Quality is the Primary Technical Bottleneck:

Performance deficits are primarily driven by sub-optimal search retrieval (e.g., pulling adjacent or off-topic data chunks) rather than generative limitations. Optimizing data engineering—specifically through smaller text chunks (500–1,000 characters) and refined query parsing—yields a significantly higher return on investment than upgrading to larger LLM architectures.

Automated Evaluation Requires Human Oversight:

Utilizing an LLM as a judge to evaluate its own outputs introduces inherent self-evaluation bias. Automated scoring serves as an efficient tool for continuous scalability and monitoring, but high-stakes medical protocols must be subject to periodic human clinical review to mitigate risks to patient safety.
 

Strategic Business Recommendations



Deploy the RAG architecture strictly as a clinical decision-support application rather than an autonomous diagnostic authority.

To optimize performance and safety in a production environment, implement the following deployment framework:

Data Chunking: Partition clinical source documents into dense, precise text segments of 500–1,000 characters with an intentional 50–150 character overlap.

Retrieval Strategy:

Employ a default top-k similarity retrieval ($k=5$), integrating Maximal Marginal Relevance (MMR) to eliminate redundant information across multi-topic user queries.

Prompt Guardrails:

Enforce explicit, safety-constrained system prompts that restrict the model's output strictly to the retrieved context and mandate deference to a licensed healthcare professional.

Quality Assurance:

Maintain an automated monitoring pipeline for groundedness and relevance, establishing an automated human-review escalation path for any output scoring below acceptable quality thresholds
